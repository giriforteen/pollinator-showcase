## Design Philosophy

### Openness as Design

This experience embodies the message it teaches: **removal over addition**.

**Design principles:**
- Generous white space (breathing room is essential)
- Botanical aesthetic (organic, not digital)
- Minimal UI chrome (trust the content)
- Visual-first storytelling (words support, don't dominate)
- Earned interactions (meaningful, not busy)

### Technical Approach

- **Single HTML file** (no build process, no dependencies)
- **Pure CSS animations** (60fps, performant)
- **Progressive enhancement** (works everywhere)
- **Mobile responsive** (desktop and phone optimized)
- **Accessible** (screen reader compatible, keyboard navigation)

## Key Concepts Explored

### 1. Natural Design vs. Forced Optimization
The bee doesn't try to save the desert. The bee seeks nectar. Pollination is the side effect.

### 2. What Was Removed
The flowers still bloom. You're still functional. But something's missing. The desert faded not from lack of water, but from lack of connection.

### 3. Restoration vs. Intervention
They didn't fix the desert. They restored the conditions where the desert could fix itself.

### 4. Cascade Effects
When the bees returned, so did the butterflies. When the butterflies returned, so did the birds. One pollinator catalyzed cascading restoration.

## Deployment

### GitHub Pages Setup

1. **Enable GitHub Pages:**
   - Go to repository Settings
   - Navigate to Pages
   - Source: Deploy from `main` branch, `/ (root)`
   - Save

2. **Custom Domain (Optional):**
   - Add `CNAME` file with your domain
   - Configure DNS: `CNAME` record pointing to `yourusername.github.io`
   - Enable HTTPS in GitHub Pages settings

3. **Access:**
   - Default: `yourusername.github.io/pollinator-showcase`
   - Custom: `pollinator.forteen.in` (or your chosen domain)

### Local Development

```bash
# Clone the repository
git clone https://github.com/yourusername/pollinator-showcase.git
cd pollinator-showcase

# Open in browser (no build needed!)
open index.html

# Or use a simple server
python3 -m http.server 8000
# Visit: http://localhost:8000
```
